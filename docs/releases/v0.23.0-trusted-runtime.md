# Mupot v0.23.0 - Trusted Runtime

## Goal

Ship Mupot `v0.23.0 - Trusted Runtime` as a dependable self-hosted agent
control plane: an operator can deploy it, connect a real agent, grant limited
authority, run and approve work, verify the result in GitHub, and recover the
system without losing state.

This release is not a marketplace, economy, new-department, full SOS-retirement,
GCP portability, or autonomous-brain expansion release. Those remain explicitly
outside the `v0.23.0` gate.

## Non-Negotiable Gates

The release is not ready until all of these are true:

- #274 has a real Hadi-host exported receipt bundle attached or linked, including
  `manifest.json`, `cutover-gate.json`, `export-receipt.json`, and
  `manifest-check.json`.
- #150 proves one complete board to task to agent to GitHub PR cycle.
- #277 has no unresolved P0/P1 security findings.
- A staging rehearsal proves upgrade, backup, restore, rollback, Queue/DLQ
  behavior, and failure reporting.
- `v0.23.0-rc.1` completes a seven-day production soak with no lost work,
  duplicated effects, unauthorized actions, or critical failures.
- `package.json`, `CHANGELOG.md`, `ROADMAP.md`, the GitHub milestone, the Git
  tag, and the GitHub Release all name the same final version.

## Current Merge Gates

Before any `v0.23.0` release candidate:

```bash
npm audit --audit-level=high
npm run typecheck
npm test
node --test fleet-runtime/*.test.mjs
bash scripts/ci-local-evidence.sh
npx wrangler deploy --dry-run --config wrangler.example.toml
```

GitHub Actions must also be green for:

- `build`
- `plugin`
- `no-secrets`
- `local-evidence`
- CodeQL analyses

## Release Order

1. Keep PR #278 green and merge the security/CI foundation.
2. Complete #274 on the real Hadi host and attach the exported evidence bundle.
3. Complete #150 with the real GitHub PR artifact linked back to the Mupot task.
4. Run the staging recovery rehearsal and save evidence in the release issue.
5. Cut `v0.23.0-rc.1`, start the seven-day soak, and log each task cycle.
6. After soak, align release metadata: package version, changelog, roadmap,
   milestone, Git tag, and GitHub Release.
7. Publish `v0.23.0` only if the completion audit proves every objective above.

## Deferred Explicitly

Do not expand this release with:

- marketplace or economy expansion
- new departments beyond what is needed for the trusted runtime proof
- full SOS retirement
- GCP or non-Cloudflare portability
- autonomous-brain expansion
